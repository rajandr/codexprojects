Export LC + XL Bill Lifecycle

Backward-compatible view: the LC version column is named VERSION. Bills use CLAIM_ID plus CLAIM_VERSION.

INSTRUMENT has INSTRUMENT_ID, VERSION, LATEST_LIFECYCLE_ID, REFERENCE_NO, BRANCH_NAME.
INSTRUMENT stays visible in both Before and After control.
INSTRUMENT_EVENTS is a post-control history table for the EL and all XL claims.
Before/After Control changes how attribute tables show lifecycle rows vs copied instrument snapshot rows.
After Control expands the view and shows the copied INSTRUMENT snapshot rows.
VERSION is the Export LC version. CLAIM_VERSION is the bill version.

UI Fetch Rules

Use these query patterns when the UI needs to decide whether to show Export LC data, XL bill data, or a pending lifecycle transaction.

Best place to explain screen loading

Current EL Screen

select *
from instrument
where instrument_id = :instrument_id;

-- Then use INSTRUMENT.VERSION
select *
from lc_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id
  and claim_id is null
  and version = :version;

select *
from party_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id
  and party_scope = 'EL'
  and claim_id is null
  and version = :version;

Selected XL Bill Screen

-- List bills under the EL
select distinct claim_id
from bill_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id;

-- Open selected bill
select max(claim_version)
from bill_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id
  and claim_id = :claim_id;

select *
from bill_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id
  and claim_id = :claim_id
  and claim_version = :claim_version;

select *
from party_details
where link_type = 'INSTRUMENT'
  and link_id = :instrument_id
  and party_scope = 'XL'
  and claim_id = :claim_id
  and claim_version = :claim_version;

Pending Lifecycle Screen

select *
from life_cycle
where lcy_id = :lifecycle_id;

-- EL work-in-progress
select *
from lc_details
where link_type = 'LIFE_CYCLE'
  and link_id = :lifecycle_id
  and claim_id is null;

-- XL work-in-progress
select *
from bill_details
where link_type = 'LIFE_CYCLE'
  and link_id = :lifecycle_id
  and claim_id = :claim_id;

Timeline / History View

select *
from instrument_events
where instrument_id = :instrument_id
order by event_seq;

-- EL-only history
select *
from instrument_events
where instrument_id = :instrument_id
  and claim_id is null
order by event_seq;

-- XL bill history
select *
from instrument_events
where instrument_id = :instrument_id
  and claim_id = :claim_id
order by claim_version;
Simple UI rule: EL tabs read rows where CLAIM_ID is null. XL tabs read rows where CLAIM_ID is filled. Controlled screens use LINK_TYPE = INSTRUMENT; editable screens use LINK_TYPE = LIFE_CYCLE. History screens use INSTRUMENT_EVENTS.
